Repo License Info is a documentation-style repo about common open-source licenses. It compares licenses such as MIT, Apache, GPL, LGPL, AGPL, BSD, MPL, EPL, Unlicense, and CC0.

The important idea is that a license is a design choice. It shapes how other people can use, modify, redistribute, and integrate a project.

## License Families

Permissive licenses such as MIT, BSD, and Apache are friendly to broad reuse. They usually require attribution but allow use in closed-source products.

Copyleft licenses such as GPL require derivative works to preserve similar freedoms. Stronger network-oriented licenses such as AGPL also consider software used over a network.

Public-domain-style options such as CC0 and Unlicense aim to remove restrictions, though legal treatment can vary across jurisdictions.

## Practical Considerations

Choosing a license depends on the project goal. A portfolio demo, a library, a commercial tool, and a collaborative research project may need different terms.

The repo is a useful learning note, but license decisions for production or company work should always be checked carefully.

## Knowledge Points

- A repo without a license is not automatically free for others to reuse.
- Permissive licenses optimize for adoption and integration.
- Copyleft licenses optimize for preserving openness in derivative works.
- Apache 2.0 includes explicit patent-related language, which can matter for companies.
- AGPL is important when software is primarily delivered as a network service.
- License compatibility can affect whether dependencies can be combined.
- Documentation repos should distinguish learning notes from legal advice.

## Takeaway

This repo captures a practical software engineering topic that is easy to overlook. Code is technical, but reuse is legal and social too.
